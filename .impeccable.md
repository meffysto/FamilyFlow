# Impeccable — FamilyFlow

## Design Context

### Users
Familles avec parents de **30-40 ans** et enfants en bas âge (nourrissons → primaire). L'app s'ouvre plusieurs fois par jour dans des contextes chaotiques : le matin en préparant le petit-déjeuner, entre deux tâches, le soir sur le canapé avec un enfant sur les genoux, au milieu de la nuit pour logger un bibi. Usage principal par les parents — les enfants interagissent via les profils thématiques (loot boxes, mascotte ferme), mais ne pilotent pas l'app.

Le job-to-be-done : **garder le contrôle de la vie familiale sans stress, en transformant l'intendance quotidienne (tâches, repas, RDV, journal bébé) en un rituel partagé et vivant**. Les données vivent en local dans un vault Obsidian — zéro backend, zéro tracking.

### Brand Personality
**Chaud · Familier · Artisanal.**

L'app doit évoquer une maison habitée plutôt qu'un tableau de bord. Référence tonale : une cuisine où mijote quelque chose, un carnet en cuir posé sur une table en bois, un potager qu'on soigne ensemble. Surtout pas : productivité corporate, SaaS générique, gamification agressive type Duolingo.

**Émotion cible** : sérénité (« tout est sous contrôle, rien ne se perd ») + une pointe de **combat joyeux** (« on s'attaque aux tâches ensemble, on gagne des trophées, la ferme pousse »). Les deux cohabitent — le quotidien devient un terrain de jeu coopératif sans jamais basculer dans l'enfantin tape-à-l'œil.

### Aesthetic Direction
Cohérence obligatoire avec le site marketing [meffysto.github.io/FamilyFlow](https://meffysto.github.io/FamilyFlow/) — l'app est le prolongement natif de cet univers.

**Palette de référence (site)** — à adapter aux contraintes mobile, pas à copier pixel près :
- Fonds : `#EDEAE4` (lin), `#F5EFE5` (crème chaude), `#FFF8EC` (parchemin clair), `#F5E6C8` (parchemin foncé)
- Bois/terre : `#6B4226` (soil), `#8B6914`, `#A0784C`, `#C4A265` (highlights)
- Vert ferme : `#6AAE4E` (primary), `#4E8A3A` (deep), `#7EB34A` (grass)
- Or/miel : `#E8C858`, `#FFD700`, `#FFF4DA`
- Textes bleu nuit profond (jamais noir pur) : `#1A1A2E`, `#3D3D56`, `#6B7280`
- Nuit : `#1D2438` (fond dark mode — bleu nuit chaud, pas noir)

Les 9 thèmes de profil enfant (voitures rouge, licorne rose, espace bleu, pirates ambre…) se superposent à cette base comme des **accents** — ils ne remplacent pas la palette artisanale, ils la teintent.

**Typographie** (reprise du site pour continuité de marque — on assume l'écart avec la règle anti-reflex d'Impeccable au nom de la cohérence brand) :
- **Display** : DM Serif Display — titres section, nombres XP, moments cérémoniaux (loot opening, passage de niveau)
- **Body** : Plus Jakarta Sans — 95 % du texte UI
- **Manuscrit** : Caveat — accents chaleureux, signatures, légendes émotionnelles (« souvenir du 12 mars », « message de Maman »), JAMAIS en body ou en labels fonctionnels
- **Mono** : JetBrains Mono — timestamps, codes, métadonnées techniques

**Motifs visuels qui font l'identité** :
- Textures parchemin/papier légères (jamais criardes)
- Bordures bois sur les zones cérémoniales (loot, ferme, récompenses)
- Pixel art assumé pour la ferme/mascotte — crisp, DB16 palette cohérente
- Atmosphère saisonnière : l'app sait qu'on est en hiver ou en été (accent subtil, pas envahissant)
- Dualité **parchemin clair ↔ bleu nuit étoilé** pour light/dark, plutôt que le classique gris clair / gris foncé

**Anti-références** (ce qu'on N'EST PAS) :
- Material Design plat, iOS stock épuré à la Notion/Linear
- Cards blanches empilées avec ombres grises — on préfère fonds teintés et séparateurs chauds
- Gamification Duolingo/Headspace — trop saturée, trop mascotte géante
- Dashboards SaaS corporate — pas de « metrics hero », pas de sparklines décoratives
- Dark mode cyan-on-noir, gradients violet-bleu, glassmorphism — bannis

### Design Principles

1. **La maison habitée, pas le tableau de bord.** Chaque écran doit sentir l'usage quotidien d'une famille, pas la démo produit. Préférer les fonds teintés au blanc pur, les séparateurs chauds aux lignes grises, les mots tendres aux labels froids.

2. **Sérénité d'abord, jeu en surprise.** L'état par défaut est calme et lisible — un parent fatigué à 22h doit s'y retrouver en 2 secondes. Les moments ludiques (loot box, ferme qui pousse, confettis XP) sont des **récompenses** qui surgissent, pas le vernis permanent de l'interface.

3. **Le français comme matière.** La langue est l'ingrédient chaud principal. Labels courts mais humains (« Ta journée », pas « Dashboard »), messages d'état qui parlent comme un proche, erreurs qui ne culpabilisent jamais. Caveat pour les moments qui méritent une voix manuscrite.

4. **Cohérence marque > règles génériques.** Quand Impeccable propose une règle (« jamais DM Serif Display », « pas de gradient »…) qui contredit l'ADN du site marketing, la marque gagne. Les règles sont appliquées partout ailleurs, sans exception.

5. **L'artisanat dans les détails, pas dans le décor.** Un coin arrondi choisi (pas 12px par défaut), une ombre chaude (pas `rgba(0,0,0,0.1)`), un easing qui respire, un motif parchemin à 3 % d'opacité — c'est là que se loge l'âme. Pas dans un gros bandeau « artisanal » collé en haut.

6. **Mobile d'abord, iPad respecté.** maxWidth 700px au-delà, pas plus. Les gestes priment sur les boutons quand c'est naturel, mais jamais de swipe dans une ScrollView (voir CLAUDE.md). Touch targets ≥ 44px, toujours.

### Accessibility Baseline
- Contraste WCAG AA minimum sur tout texte ≥ 14px, AAA visé sur body lecture
- Reduced motion respecté (`useReducedMotion()` depuis Reanimated) — les animations ferme/loot ont une version statique
- Dynamic Type iOS supporté (tailles relatives, pas de `fontSize: 15` qui ignore les préférences système)
- Les 9 thèmes profil doivent tous passer le contraste sur fond clair ET sombre — c'est pour ça qu'il existe `primaryDark` dans `themes.ts`
