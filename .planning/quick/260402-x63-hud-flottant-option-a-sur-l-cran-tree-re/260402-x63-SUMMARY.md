# Quick 260402-x63: HUD Flottant Absolute sur l'ecran Tree — Summary

## One-liner

HUD ferme rendu en overlay absolu (top: insets.top, zIndex: 10) sur l'ecran arbre avec fond semi-transparent et ScrollView decale via paddingTop pour effet diorama immersif.

## What Was Built

Le bloc `farmHud` (coins, streak, cultures, saison) a ete deplace hors du flux normal du ScrollView pour flotter en position `absolute` par-dessus le diorama pixel. Le `SafeAreaView` wrapper a ete remplace par un `View` ordinaire car le HUD gere lui-meme le decalage `top` via `useSafeAreaInsets`. Le `ScrollView` compensevec `paddingTop: insets.top + 44` pour que le contenu ne se retrouve pas masque derriere le HUD.

## Decisions Made

- **SafeAreaView → View** : Supprimer le safe area automatique du wrapper car le HUD gere le top inset manuellement via `useSafeAreaInsets().top`. Cela permet au fond de l'ecran d'etre transparent — le monde pixel est visible derriere.
- **position absolute + zIndex 10** : Le HUD est rendu apres le `ScrollView` dans le JSX pour garantir le z-order sans dependre des styles Shadow (iOS n'eleve pas automatiquement les views en dehors du flux).
- **paddingTop: insets.top + 44** : Valeur 44 correspond a `paddingVertical: Spacing.sm * 2 + hauteur ligne ~20 + marge`. Le contenu du scroll commence sous le HUD.
- **rgba sans BlurView** : Le BlurView a un cout GPU non negligeable. Le fond rgba(0,0,0,0.55) / rgba(255,255,255,0.7) est visuellement suffisant et plus performant.
