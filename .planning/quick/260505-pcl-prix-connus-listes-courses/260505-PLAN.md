# Objectif

Permettre à l'utilisateur d'ajouter ou ajuster manuellement le prix d'un article dans les listes de courses, en complément du prix auto-dérivé depuis les BudgetEntries. Les prix manuels sont stockés dans un pricebook persistant (vault) pour propager automatiquement sur tous les items futurs portant le même nom canonique.

# Contexte

- Système existant : `lib/courses-prices.ts` calcule un prix par article via 2 passes (canonical strict → fuzzy fallback) sur les BudgetEntries des catégories courses/bébé/maison. Affiché lecture-seule dans `app/(tabs)/meals.tsx` avec préfixe `≈` (high) ou `~` (low).
- Limite : pas d'override manuel possible, et pas de prix affiché si jamais aucun match dans le budget.
- Décision design : le pricebook gagne sur le budget — déclaration explicite utilisateur > inférence statistique.

# Format vault

Fichier : `04 - Budget/Prix connus.md`

```markdown
---
updatedAt: 2026-05-05
---
# Prix connus

- Pain: 1,95
- Lait demi-écrémé 1L: 0,95
```

Round-trip Obsidian propre. Clé de match = `parseCanonical(label)` côté lookup (les deux côtés tokenisés et comparés set-equal).

# Plan

1. **lib/parser.ts** — `parsePriceBook(content)` retourne `{ updatedAt, entries: Array<{ label, canonical, price }> }`, et `serializePriceBook(state)` réécrit le fichier. Tolère prix avec virgule FR ("1,95") ou point ("1.95"), espace optionnel avant `€`. Ignore lignes mal formées sans crash.

2. **lib/courses-prices.ts** — étendre `CoursePriceInfo['confidence']` avec `'manual'`. `getLastPriceFor(articleName, entries, priceBook?)` : si `priceBook` fourni et match canonical exact (set-equal des tokens via `parseCanonical`), retourne `{ price, daysAgo: 0, stale: false, confidence: 'manual', sampleSize: 1 }` immédiatement. Sinon comportement actuel inchangé. `computeRemainingEstimate(items, entries, priceBook?)` ajoute le param.

3. **hooks/useVaultPriceBook.ts** (nouveau) — `usePriceBook(vault)` hook : charge `04 - Budget/Prix connus.md` au boot (crée si absent), expose `{ priceBook: Map<canonical, { label, price }>, setPrice(label, price), removePrice(canonical), refresh() }`. Pattern enqueueWrite identique à `useVaultCourses` (file série pour éviter les races).

4. **hooks/useVault.ts** — intégrer dans `useVaultInternal()` : appel du hook, exposition dans le retour et le type `VaultContextValue`.

5. **app/(tabs)/meals.tsx** — brancher `priceBook` :
   - injecté dans `coursePriceByItemId` (param 3) et `courseRemainingEstimate` (param 3)
   - bloc d'affichage prix (l. ~1854-1872) : devient `TouchableOpacity`
     - tap → `Alert.prompt` (iOS-only, fallback message Android) avec keyboard numeric → parse virgule/point → `setPrice(item.text, price)`
     - long-press si confidence manual → Alert "Effacer le prix connu" → `removePrice(canonical)`
   - si `priceInfo === null` : afficher slot "+ €" tappable au même endroit (même handler tap)

6. **Display différencié** :
   - prix `'manual'` : couleur `colors.primary`, `fontWeight: bold`, **pas de préfixe** (juste "1,95 €")
   - prix `'high'`/`'low'` (auto) : inchangé (textMuted + préfixe `≈`/`~`)

7. **Tests** — `lib/__tests__/parser.test.ts` : describe('parsePriceBook') :
   - parse vide (frontmatter only) → entries: []
   - parse avec virgule FR ("Pain: 1,95") → entry { label: 'Pain', canonical: ['pain'], price: 1.95 }
   - parse tolère "Pain: 1,95 €" et "Pain : 1,95" (espace before colon)
   - roundtrip serialize → parse identité (label, price)
   - lignes invalides ignorées sans crash

8. **i18n** — `locales/fr/common.json` et `locales/en/common.json`, sous `meals.shopping.*` :
   - `priceManualPrompt` : "Prix de {{name}}" / "Price for {{name}}"
   - `priceManualMessage` : "Saisis le prix en €" / "Enter price in €"
   - `priceManualRemove` : "Effacer le prix connu" / "Remove known price"
   - `priceManualRemoveConfirm` : "Effacer" / "Remove"
   - `priceAddSlot` : "+ €"
   - `priceManualSave` : "OK"
   - `priceManualCancel` : "Annuler" / "Cancel"

# Risques

- `Alert.prompt` est iOS-only — sur Android, fallback dégradé (toast + ouvrir l'éditeur complet) ou simple modal custom. Comme l'app est iOS-first (TestFlight), on accepte le no-op Android pour v1.
- Si l'utilisateur tape un prix invalide ("12,5,5"), le parse retourne NaN — on affiche un Alert d'erreur sans crasher.
