## Helpers exportés (lib/story-books.ts)

- `groupStoriesByBook(stories)` : regroupe par `livreId` ; histoires sans `livreId` deviennent des livres implicites mono-chapitre (`isImplicit=true`). Tri global descendant par date du dernier chapitre.
- `getBookCasting(chapters)` : union ordonnée des slugs personnages (premier chapitre d'apparition d'abord).
- `getNextChapterNumber(chapters)` : `max(chapitre) + 1`, ou `2` si aucun chapitre numéroté (= legacy implicite).
- `buildBookContextForPrompt(book)` : construit le payload `book` pour `generateBedtimeStory` (texte intégral chap N-1 + résumés antérieurs + lockedCasting + numéro chapitre).
- `slugifyBookTitle(titre)` : lowercase + strip diacritics NFD + `[a-z0-9]+ → '-'` + max 60 chars + suffixe `-<6 chars Date.now base36>` pour unicité.

## Comportement du prompt Claude

### Sans `book` ni `trancheAge` (legacy strict)
Prompt **identique à l'ancien**. Seule différence : le JSON output inclut désormais TOUJOURS `memorySummary` (Claude ajoute un résumé neutre 4-5 phrases ; persisté au frontmatter). Pas de breaking change pour les générations existantes.

### Avec `trancheAge` seul (1er chapitre)
La ligne `Vocabulaire adapté à <âge>` est REMPLACÉE par les règles tranche :
- **3-5** : phrases 8-12 mots, vocabulaire concret, émotions de base, pas d'abstraction.
- **6-8** : phrases 12-18 mots, concepts narratifs simples (amitié/courage), petits dilemmes résolus.
- **9+** : phrases 15-25 mots, thèmes plus profonds (responsabilité/identité), sous-texte autorisé.

### Avec `book` (chapitre N>=2)
Prompt enrichi de 2 blocs :

1. **CASTING VERROUILLÉ + CASTING DISPONIBLE** (remplace `buildCharactersPromptSection` quand multi-voix actif). Les personnages déjà introduits doivent garder leur personnalité ; max 1 nouveau par chapitre.
2. **MÉMOIRE NARRATIVE** : texte intégral du chapitre N-1 + résumés courts des chapitres antérieurs + règles de continuité (pas de récap au début, fin apaisée, cohérence personnages).

`max_tokens` budget : `+400` quand `book` présent pour absorber `memorySummary` sans tronquer.
