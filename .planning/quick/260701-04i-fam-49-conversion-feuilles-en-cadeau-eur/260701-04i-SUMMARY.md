# Quick Task 260701-04i : FAM-49 conversion feuilles 🍃 → cadeau € Summary

Système configurable par famille permettant à l'enfant de convertir ses feuilles 🍃 en crédit cadeau €, avec déduction immédiate (miroir PlotUpgradeSheet), file d'attente parentale persistée dans le vault, et remboursement sur refus. Feature OFF par défaut.

## Ce qui a été construit

- **Task 1 — Fondations** : type `GiftRequest` (`pending`/`approved`/`rejected`) dans `lib/types.ts` ; `GamificationConfig.giftExchange` (toggle + leavesCost + euroValue, OFF par défaut) dans `rewards.ts` ; paire `parsePendingGifts`/`serializePendingGifts` round-trip dans `lib/parser.ts`.
- **Task 2 — Logique + UI parent** : hook isolé `useGiftExchange` (`requestExchange`/`approveGift`/`rejectGift`/`loadAll`) répliquant le bloc deductCoins/addCoins de useFarm ; carte config (Switch + 2 TextInput ratio) et carte « Cadeaux en attente » (Valider/Refuser) dans `SettingsGamification.tsx` ; clés i18n FR/EN.
- **Task 3 — UI enfant** : `GiftExchangeSheet` (sheet farm cozy miroir de PlotUpgradeSheet) ; chip « Échanger » gatée (feature ON + rôle enfant/ado) et rendu du sheet dans `tree.tsx` ; clé `mascot.actions.echanger` FR/EN.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Round-trip parser instable sur resolvedAt vide**
- **Found during:** Task 1 (vérification round-trip)
- **Issue:** `parsePendingGifts` faisait `line.trim()` puis `split(' | ')`. Pour une demande `pending` (resolvedAt vide), la ligne sérialisée se termine par `| ` (trailing space) ; le `trim()` le supprimait, réduisant la ligne à 7 colonnes → la demande était silencieusement perdue au reload (les demandes pending ne survivaient donc PAS au relaunch — must_have cassé).
- **Fix:** split sur `|` brut + `.map(c => c.trim())` par cellule (robuste, profileName déjà échappé). Round-trip confirmé stable sur fixture 2 lignes (pending + rejected).
- **Files modified:** lib/parser.ts
- **Commit:** 7eebdc8a

### i18n placement (clarification, pas une déviation fonctionnelle)
Le plan mentionnait `locales/fr/gamification.json`, mais `t('settings.gamification.*')` et `t('mascot.actions.*')` résolvent contre le namespace par défaut **`common`** (cf. `lib/i18n.ts` `defaultNS: 'common'`, et les clés existantes vivent dans `common.json`). Les nouvelles clés ont donc été ajoutées dans `locales/{fr,en}/common.json` pour qu'elles résolvent correctement. Les messages du sheet sont en français littéral (miroir de PlotUpgradeSheet, qui ne passe pas par i18n).
