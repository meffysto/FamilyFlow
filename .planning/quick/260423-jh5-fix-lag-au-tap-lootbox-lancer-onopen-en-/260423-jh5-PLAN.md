# Context technique
La fonction `handleOpen` (lignes 757-926) a actuellement ce flow sérialisé :
1. `setPhase('spinning')` — instant
2. Animations shake lancées (non-bloquantes, réanimées)
3. `await new Promise(r => setTimeout(r, 600))` — BLOQUE 600ms AVANT le fetch
4. `const box = await onOpen()` — BLOQUE pendant l'appel API (100-500ms typique)
5. `await new Promise(r => setTimeout(r, 500))` — BLOQUE 500ms supplémentaires pour rien
6. Pour mythique : 3×`await Haptics.impactAsync()` + 2× `await setTimeout(250-300ms)` séquentiels — BLOQUE ~750ms AVANT le reveal visuel
7. Pour légendaire : 2×`await Haptics.impactAsync()` + 1× `await setTimeout(200ms)` — BLOQUE ~200ms

Total worst-case (mythique) avant reveal : 600 + 500 + 750 = 1850ms de bloquants purs (hors API).
