# Quick Task Q49: Échange inter-familles via le Port du village

**One-liner:** Système de colis inter-familles sans backend — code-cadeau base36 FF-{payload} partageable via share sheet native, validé au Port (palier 12000).

## Objectif

Ajouter une mécanique sociale entre familles utilisant l'app : le Port village (bâtiment palier 12000) permet d'envoyer et recevoir des colis contenant des items consommables (village/ferme/récoltes) via un code partageable. Zéro backend — le code encode le contenu complet.

## Ce qui a été livré

### Task 1 — Moteur pur `trade-engine.ts`

Crée `lib/village/trade-engine.ts` — module pur TypeScript, zéro import hook/context :

- `encodeTrade(payload)` : code `FF-{catChar}{itemId8}{qty2}{ts6b36}{nonce3b36}{checksum2b36}` (22 chars après préfixe)
- `decodeTrade(code)` : validation format + checksum, retourne null si invalide
- `isTradeExpired(payload)` : true si > 48h depuis timestamp epoch
- `isTradeAlreadyClaimed(code, claimedCodes)` : lookup liste locale
- `canSendTradeToday` / `incrementTradesSent` / `tradesRemainingToday` : anti-abus journalier (pattern identique gift-engine)
- `getAvailableTradeItems(category, ...)` : liste items dispo par catégorie avec badge stock
- `getTradeItemMeta(category, itemId)` : label + emoji pour affichage réception
- Champs `trade_claimed_codes` et `trade_sent_today` ajoutés à `FarmProfileData` (lib/types.ts)
- Export via barrel `lib/village/index.ts`

### Task 2 — Modales UI

**`PortTradeModal.tsx`** — pageSheet, 2 onglets :
- Onglet Envoyer : badge envois restants, sélecteur catégorie (Village/Ferme/Récoltes), grille items avec badges stock, stepper quantité, code copyable + bouton Partager (RNShare)
- Onglet Recevoir : TextInput code FF-..., messages d'erreur inline, bouton Ouvrir le colis

**`TradeReceiptModal.tsx`** — Clone simplifié de GiftReceiptModal :
- Animation spring bounce (translateY + scale), pulse packageScale + confetti + haptic
- Affiche emoji + label + quantité reçus, tag "Via le Port du village"

### Task 3 — Intégration

**`useGarden.ts`** (nouveaux exports dans UseGardenReturn) :
- `sendTrade(category, itemId, qty, profileId)` : vérifie quota, déduit stock (village → jardin-familial.md, farm/harvest → gami-{id}.md), incrémente trade_sent_today, encode code
- `receiveTrade(code, profileId)` : décode + valide expiration + anti-double-claim, ajoute item, archive code dans trade_claimed_codes (max 200 entrées)
- `canSendTradeToday` et `tradesSentRemaining` dérivés via useMemo
- useEffect charge trade_sent_today au mount du profil actif

**`lib/parser.ts`** : parseFarmProfile + serializeFarmProfile supportent `trade_claimed_codes` (CSV) et `trade_sent_today`

**`app/(tabs)/village.tsx`** :
- `handleBuildingPress` intercepte `buildingId === 'port'` → `setShowPortTrade(true)`, autres bâtiments inchangés
- `loadFarmInventories` charge farmInventory + harvestInventory du profil actif, refresh après chaque trade
- `<PortTradeModal>` et `<TradeReceiptModal>` câblés avec state et callbacks
