## Spacing Scale

Tokens existants dans `constants/spacing.ts` — utiliser UNIQUEMENT ces valeurs, jamais de valeurs numériques brutes.

| Token | Value | Usage dans cette phase |
|-------|-------|----------------------|
| `Spacing.xs` | 4px | Gap icône-texte dans les badges de sévérité, padding horizontal badge inline ingrédient |
| `Spacing.md` | 8px | Gap entre chips dans la rangée de préférences, padding badge compact |
| `Spacing.xl` | 12px | Padding interne des sections CollapsibleSection, gap entre sections |
| `Spacing['2xl']` | 16px | Padding horizontal écran (contentContainer), margin entre profils |
| `Spacing['3xl']` | 20px | Padding modal pageSheet (sélecteur convives, preview vocale) |
| `Spacing['4xl']` | 24px | Padding vertical safe area, espacement entre section Famille / section Invités |
| `Spacing['5xl']` | 32px | Espacement majeur entre le bandeau allergie et le contenu recette |

Exceptions:
- Touch targets sur bouton micro flottant (DictaphoneRecorder) et bouton "+" d'ajout : minimum 44px de zone de tap (peut utiliser paddingVertical compensatoire sans modifier le token).
- Border radius: `Radius.full` (9999) sur les chips préférences, `Radius.lg` (12) sur le bandeau allergie, `Radius.xs` (4) sur les badges inline ingrédients.

**Note sur les tokens non-standard:** `Spacing.xl` (12px) et `Spacing['3xl']` (20px) sont des tokens pré-existants dans `constants/spacing.ts` — ils ne suivent pas strictement l'échelle canonique 8-pt mais sont justifiés car ce sont des valeurs projet établies. Exception documentée, pas une dérogation ponctuelle.

## Color

Utiliser TOUJOURS `useThemeColors()` — jamais de valeurs hex codées en dur (sauf exception délibérée documentée ci-dessous).

| Role | Token light | Token dark | Usage |
|------|------------|-----------|-------|
| Dominant (60%) | `colors.bg` `#EDEAE4` | `#12151A` | Fond écran Préférences alimentaires |
| Secondary (30%) | `colors.card` `#FFFFFF` | `#1C1F28` | Cards profil, sections CollapsibleSection, fond modal pageSheet |
| Accent (10%) | `primary` (via `useThemeColors()`) | — | Bouton "+" ajouter préférence, état selected des Chips, border active du champ autocomplete |

**Couleurs sémantiques de sévérité — mapping sur tokens existants:**

| Sévérité | Background token | Text token | Border | Usage |
|----------|-----------------|-----------|--------|-------|
| `allergie` (P0 SAFETY) | `colors.errorBg` | `colors.errorText` | `colors.error` | Bandeau allergie non-dismissible + badge inline rouge sur ingrédient conflictuel |
| `intolerance` | `colors.warningBg` | `colors.warningText` | `colors.warning` | Bandeau intolérance + badge inline orange |
| `regime` / `aversion` | `colors.tagMention` / `colors.infoBg` | `colors.tagMentionText` / `colors.info` | `colors.warning` | Bandeau jaune/info + badge inline jaune |

**P0 SAFETY — contrainte couleur allergie:**
Le bandeau `allergie` utilise `colors.errorBg` + `colors.errorText` + border latérale `colors.error` (3px) en permanence. Aucune prop de couleur conditionnelle ne peut rendre ce bandeau invisible ou neutral.

Accent réservé pour: bouton "+" ajouter préférence, Chip état selected, border active champ autocomplete, indicateur de progression dans la modale preview vocale. PAS utilisé pour les badges de sévérité (ceux-ci ont leurs propres tokens sémantiques).

Destructive: `colors.error` — suppression d'une préférence (swipe-to-delete via `ReanimatedSwipeable`) + suppression d'un invité.

## Composants et états par écran

### Écran principal: Préférences alimentaires (`app/dietary.tsx`)

**Point focal / hiérarchie visuelle:** Point focal primaire: badge allergie dans la première `ProfileFoodCard` — poids visuel maximal (couleur `colors.errorBg`, border latérale 3px `colors.error`). Toutes les autres informations sont visuellement secondaires par rapport à ce badge. La hiérarchie est: titre écran > avatar+nom profil > badge allergie > chips intolérance > chips régime/aversion.

**Structure:**
```
ScrollView (contentContainerStyle avec Layout.contentContainer)
  ├── Section "Membres famille" (titre H2)
  │   └── Pour chaque membre: ProfileFoodCard
  │       ├── Avatar + nom (row)
  │       └── CollapsibleSection × 4 (Allergies / Intolérances / Régimes / Aversions)
  │           ├── Chips existantes + bouton "+" pour ajouter
  │           └── TextInput autocomplete (si catalogue) ou texte libre (aversions)
  └── Section "Invités récurrents" (titre H2, séparation visuelle Spacing['4xl'])
      ├── Bouton "Ajouter un invité" (Button variant outline)
      └── Pour chaque invité: même structure que ProfileFoodCard
```

**Bouton micro flottant:** DictaphoneRecorder positionné dans le header (right side), zone de tap 44px minimum.

**Accessibilité:** Le bouton "+" d'ajout de préférence doit porter `accessibilityLabel="Ajouter une préférence alimentaire"` (plus descriptif que le seul caractère "+").

**États:**
- Vide famille (aucune préférence aucun membre): afficher CollapsibleSection fermée avec hint "Aucune allergie enregistrée — appuyez + pour ajouter"
- Vide invités: message inline "Aucun invité récurrent enregistré" + bouton "Ajouter un invité"
- Chargement: aucun skeleton — les données vault sont synchrones

### Autocomplete dropdown

**Comportement:** Apparaît sous le TextInput, max 5 entrées visibles, scroll interne si plus. Fond `colors.card`, border `colors.border`, border-radius `Radius.lg`. Chaque item: padding `Spacing.xl` vertical, texte `FontSize.body`.

**Accessibilité:** `accessibilityRole="list"` sur le container, `accessibilityRole="button"` sur chaque item.

### Bandeau conflit recette (dans l'écran détail recette existant)

**Position:** En tête de l'écran recette, avant tous les ingrédients. Toujours rendu (pas de conditional return).

**Structure bandeau:**
```
AllergenBanner (View, pointerEvents sur zone swipe = 'none')
  ├── Ligne allergie: fond errorBg, texte errorText, border gauche 3px error — TOUJOURS visible, non-collapsible
  ├── Ligne intolérance: fond warningBg — collapsible si aucune allergie, non-collapsible si allergie présente
  └── Ligne régime/aversion: fond tagMention/infoBg — collapsible
```

**Règle d'affichage:** Si `severity === 'allergie'` présente → toutes les lignes sont rendues expanded et non-collapsibles. Si seulement `intolerance` → ligne intolerance expanded, non-collapsible ; ligne régime collapsible. Si seulement `regime`/`aversion` → collapsible.

**Badge inline ingrédient:** `Badge` component (existant), variant mappé sur sévérité la plus haute pour cet ingrédient, taille `sm`, positionné après le texte de l'ingrédient.

### Bouton "Vérifier les conflits pour…" (dans écran recette)

Un `Button` variant `outline` sous le bandeau, label "Vérifier les conflits pour…" → ouvre modal `pageSheet`.

### Modal sélecteur convives (pageSheet)

Chips multiselect (composant `Chip` existant) pour membres famille + invités. Deux sections: "Famille" + "Invités". Bouton "Vérifier les convives" (primary) en footer de modal. Pas de persistance — résultat volatil uniquement.

### Modal preview vocale (pageSheet)

Liste des extractions IA: chaque ligne = checkbox + profil cible + catégorie + item + sévérité, tous éditables inline. Bouton "Confirmer (N)" en footer primary. Bouton "Tout décocher" en secondary. Pas de fermeture sans action explicite.

## Interaction Contracts

### Swipe-to-delete préférence (chip dans CollapsibleSection)

- Utiliser `ReanimatedSwipeable` (pas `Swipeable`) depuis `react-native-gesture-handler/ReanimatedSwipeable`
- Action droite: bouton rouge "Supprimer" (`colors.error`, texte blanc)
- Haptic: `Haptics.impactAsync(ImpactFeedbackStyle.Medium)` au moment de la suppression effective
- Pas de confirmation modale pour les préférences individuelles (réversibles)
- Confirmation via `Alert.alert` UNIQUEMENT pour la suppression d'un invité complet

### Ajout via autocomplete

- `TextInput` déclenche le dropdown dès 1 caractère tapé
- Sélection d'un item catalogue: haptic `Haptics.selectionAsync()` + chip apparaît dans la rangée + TextInput se vide
- Texte libre (hors catalogue): touche Entrée ou bouton "Ajouter une préférence" valide le texte brut comme chip

### Bandeau allergie (P0 SAFETY enforcement)

- Composant `AllergenBanner` n'expose AUCUNE prop `onDismiss`, `onClose`, ni `dismissible`
- `pointerEvents='none'` sur la View container du bandeau allergie (résistance aux gestes accidentels)
- La ligne allergie reste rendue même si `expanded=false` sur les autres lignes — elle n'est pas enfant d'un CollapsibleSection

### Modal pageSheet (sélecteur convives + preview vocale)

- Présentation: `pageSheet` + drag-to-dismiss
- Fermeture: drag down OU bouton "Fermer" (ModalHeader existant)
- Footer sticky avec bouton primary

### Animations

- `useSharedValue` + `withSpring` sur l'apparition des chips (scale 0.8 → 1.0)
- Spring config: `{ damping: 15, stiffness: 200 }` (cohérent avec les patterns existants)
- Pas d'animation sur le bandeau allergie — il doit être immédiatement visible sans délai
