- HarvestBurst particle travel radius: 25‚Äì45px (non-token range, uses randomized spread per existing pattern ‚Äî do not change)
- Touch target minimum: 44px for any interactive element in this phase (no explicit interactive elements added; all feedback is passive)

### Visual Priority ‚Äî Simultaneous Feedback Elements

When an effect fires, up to three feedback elements appear simultaneously. Their visual hierarchy is:

1. **Primary: HarvestBurst burst** ‚Äî center-screen particle overlay. Full-screen z-index. Immediate, most salient.
2. **Secondary: Toast** ‚Äî top bar notification. Appears at same instant (D-02), but confined to top edge ‚Äî does not compete with burst center.
3. **Tertiary: Companion bubble** ‚Äî updates on next `tree.tsx` render, not simultaneously with burst/toast. Lower temporal and spatial priority; user may not be on the farm screen.

Rationale: this priority order ensures the celebration moment (burst) is never visually occluded, the toast provides immediate textual confirmation, and the companion message serves as a delayed contextual reinforcement.

## Interaction States

| State | Behavior |
|-------|---------|
| Effect triggered, cap not exceeded | Toast + HarvestBurst variant + haptic fire in parallel (D-02). Companion bubble updates on next tree.tsx render with subType message. |
| Effect triggered, cap exceeded | Silent ‚Äî no toast, no burst, no haptic (D-03). Standard XP awarded. |
| No `CategoryId` matched (standard XP) | No effect toast. Existing generic `task_completed` companion message as today. |
| `isWeb` platform | Haptics skipped (guard). Toast and burst fire normally. |
| Reduced motion | Toast enters/exits without spring animation (existing `useReducedMotion` in `ToastContext`). HarvestBurst particles skip travel animation ‚Äî opacity fade only. |

## Animation Specs

All animations use `react-native-reanimated` (mandatory per CLAUDE.md).

### HarvestBurst variant additions

| Variant | Particle count | Particle size | Travel distance | Duration |
|---------|---------------|--------------|-----------------|---------|
| `ambient` (default, 8 particles) | 8 | 6px (existing) | 25‚Äì45px (existing) | 800ms (existing) |
| `rare` | 10 | 6px | 30‚Äì50px | 900ms |
| `golden` | 12 | 8px | 35‚Äì60px | 1000ms |

Spring config for particle travel (unchanged from existing): `{ damping: 8, stiffness: 80 }`

Label float for golden variant: `-60px` vertical travel over `1000ms` (vs existing `-45px` / `800ms`).
Label font size for golden: `16px` ‚Äî mapped from the previously considered 18px. At burst scale (particle overlay, full-screen context), the 2px reduction is imperceptible. Using 16px keeps the typography contract to exactly 4 sizes.

### Toast entry animation (existing ‚Äî do not change)
- Enter: `withSpring(0, { damping: 18, stiffness: 200 })` translateY + `withSpring(1, { damping: 10, stiffness: 180 })` scale
- Exit: `withTiming(-120, { duration: 250 })` + `withTiming(0, { duration: 250 })`
