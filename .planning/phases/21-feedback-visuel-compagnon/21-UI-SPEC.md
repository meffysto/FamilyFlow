| Property | Value |

## Spacing Scale

Source: `constants/spacing.ts` (pre-existing, do not redefine)

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| `Spacing.xs` | 4px | Icon-to-text gap inside toast rich layout |
| `Spacing.md` | 8px | Toast internal padding (paddingVertical already `Spacing.xl` = 12px) |
| `Spacing.xl` | 12px | Toast gap between icon and text column (toastRich `gap`) |
| `Spacing['2xl']` | 16px | Toast horizontal padding + left/right container offset |
| `Spacing['4xl']` | 24px | HarvestBurst vertical travel (label floats -45px, keep particle radius in this range) |
| `Spacing['5xl']` | 32px | Companion bubble vertical offset from farm sprite anchor |

Exceptions:
- HarvestBurst particle travel radius: 25–45px (non-token range, uses randomized spread per existing pattern — do not change)
- Touch target minimum: 44px for any interactive element in this phase (no explicit interactive elements added; all feedback is passive)

| Variant | Trigger categories | Particle color | Label color | Particle count |

- In dark mode all variants remain unchanged — they are celebratory overlays on a dark background and have sufficient contrast.

## Haptic Patterns — Contract

Source: `lib/mascot/haptics.ts` (extend, do not modify existing patterns)

New function names follow the established `haptics{Name}()` convention.

| Pattern function | Categories | Sequence | Rationale |
|-----------------|------------|---------|-----------|
| `hapticsEffectLight()` | `menage_quotidien`, `menage_hebdo` | `Light` impact once | Routine, non-disruptive |
| `hapticsEffectMedium()` | `courses`, `enfants_routines`, `enfants_devoirs`, `budget_admin` | `Light` → 80ms → `Medium` | Moderate satisfaction |
| `hapticsEffectStrong()` | `cuisine_repas`, `gratitude_famille` | `Medium` → 100ms → `Medium` → 80ms → `Success` notification | Warm celebration |
| `hapticsEffectGolden()` | `bebe_soins`, `rendez_vous` | `Light` → 100ms → `Medium` → 80ms → `Heavy` → 150ms → `Success` notification | Epic moment (mirrors `hapticsEvolution` cadence) |

Guards:
- All functions must include `if (isWeb) return;` guard (same pattern as existing haptics).
- All sequences use `async function` + `delay()` helper (same as existing).
- Do not call haptic if `isCapExceeded === true` — effect is silent (per D-03).

## Companion Message Contract

Source: `lib/mascot/companion-engine.ts` `MESSAGE_TEMPLATES` + `lib/mascot/companion-types.ts`

### Architecture (D-04, D-05, D-06)

- Add `subType?: CategoryId` to `CompanionMessageContext` in `companion-types.ts`.
- In `pickCompanionMessage`: if `context.subType` exists and `MESSAGE_TEMPLATES[task_completed_${context.subType}]` exists, pick from sub-type pool; else fall back to generic `task_completed` pool.
- The companion speaks in its bubble on `tree.tsx` only — no cross-screen toast (D-06).

### Sub-type key naming convention

Keys follow `companion.msg.taskDone.{categoryId}.{n}` — 2 templates per category = 20 new i18n keys total.

| CategoryId | i18n key base | FR template hint | EN template hint |
|------------|--------------|-----------------|-----------------|
| `menage_quotidien` | `companion.msg.taskDone.menage_quotidien` | Ref quotidien + herbes retirées | Ref daily clean + weeds |
| `menage_hebdo` | `companion.msg.taskDone.menage_hebdo` | Ref grand nettoyage + ferme réparée | Ref deep clean + farm repaired |
| `courses` | `companion.msg.taskDone.courses` | Ref courses + bâtiments turbo | Ref shopping + building boost |
| `enfants_routines` | `companion.msg.taskDone.enfants_routines` | Ref routines enfants + humeur compagnon | Ref child routines + companion mood |
| `enfants_devoirs` | `companion.msg.taskDone.enfants_devoirs` | Ref devoirs + sprint croissance | Ref homework + growth sprint |
| `rendez_vous` | `companion.msg.taskDone.rendez_vous` | Ref rdv médical + graine rare | Ref appointment + rare seed |
| `gratitude_famille` | `companion.msg.taskDone.gratitude_famille` | Ref gratitude / anniversaire + saga | Ref gratitude / birthday + saga |
| `budget_admin` | `companion.msg.taskDone.budget_admin` | Ref budget/admin + capacité bâtiment | Ref budget/admin + building capacity |
| `bebe_soins` | `companion.msg.taskDone.bebe_soins` | Ref soins bébé + récolte dorée | Ref baby care + golden harvest |
| `cuisine_repas` | `companion.msg.taskDone.cuisine_repas` | Ref cuisine + recette rare | Ref cooking + rare recipe |

### Message tone constraints (from companion-engine prompt rules)
- 1 phrase courte et naturelle — no emoji — tutoie — no quotes — no self-introduction.
- Reference the real category AND the effect triggered (both in the message).
- Vary vocabulary between the 2 templates per category.

## Interaction States

| State | Behavior |
|-------|---------|
| Effect triggered, cap not exceeded | Toast + HarvestBurst variant + haptic fire in parallel (D-02). Companion bubble updates on next tree.tsx render with subType message. |
| Effect triggered, cap exceeded | Silent — no toast, no burst, no haptic (D-03). Standard XP awarded. |
| No `CategoryId` matched (standard XP) | No effect toast. Existing generic `task_completed` companion message as today. |
| `isWeb` platform | Haptics skipped (guard). Toast and burst fire normally. |
| Reduced motion | Toast enters/exits without spring animation (existing `useReducedMotion` in `ToastContext`). HarvestBurst particles skip travel animation — opacity fade only. |

## Animation Specs

All animations use `react-native-reanimated` (mandatory per CLAUDE.md).

### HarvestBurst variant additions

| Variant | Particle count | Particle size | Travel distance | Duration |
|---------|---------------|--------------|-----------------|---------|
| `ambient` (default, 8 particles) | 8 | 6px (existing) | 25–45px (existing) | 800ms (existing) |
| `rare` | 10 | 6px | 30–50px | 900ms |
| `golden` | 12 | 8px | 35–60px | 1000ms |

Spring config for particle travel (unchanged from existing): `{ damping: 8, stiffness: 80 }`

Label float for golden variant: `-60px` vertical travel over `1000ms` (vs existing `-45px` / `800ms`).
Label font size for golden: `18px` (vs existing `16px`).

### Toast entry animation (existing — do not change)
- Enter: `withSpring(0, { damping: 18, stiffness: 200 })` translateY + `withSpring(1, { damping: 10, stiffness: 180 })` scale
- Exit: `withTiming(-120, { duration: 250 })` + `withTiming(0, { duration: 250 })`
