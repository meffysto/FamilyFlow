- Tous les SVG sont **inline** (jamais de référence externe). Tous les attributs `fill` / `stroke` doivent utiliser `opts.color` (default `terracotta` ou `sage` selon le rôle — fougères/lichens en sage, drop caps/séparateurs/numéros en terracotta).

- Chaque SVG doit être **simple, lisible, "édition collector"** — pas de complexité gratuite. Compteur cible : ~5-15 paths/circles/lines par fonction.
- Aucune dépendance npm. Ce sont des template literals SVG.
- Préfixe `<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 100 100" ...>` ou similaire.
