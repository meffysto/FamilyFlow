- `gamification.md` n'est plus jamais écrit — uniquement lu dans `migrateGamification()` comme backup legacy

## Decisions Made

- `migrateGamification()` lit `gamification.md` mais ne l'écrit jamais — garantit la backward-compatibility pour les utilisateurs existants qui ont un `gamification.md` sur leur device
- Les 2 literals `'gamification.md'` restants dans useVault.ts sont intentionnels (dans `migrateGamification()` uniquement) — zéro écriture sur ce fichier
- Merge partiel `setGamiData(prev => ...)` pour les mutations single-profil afin de ne pas écraser les données des autres profils en mémoire
- `updateProfile` renomme le contenu interne de `gami-{profileId}.md` sans changer le nom du fichier (l'ID est stable)
- `completeDefi` boucle sur les participants et écrit un fichier per-profil individuel pour chaque participant
