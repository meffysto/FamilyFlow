# Phase 08.1 Plan 01: Split Gamification Par Profil (lib/vault + hooks) Summary

**Migration gamification.md → gami-{id}.md per-profil dans vault.ts, useFarm.ts, useGamification.ts et SettingsGamification.tsx pour éliminer les conflits d'écriture iCloud multi-device**

## Performance

- **Duration:** ~15 min
- **Started:** 2026-03-30T00:00:00Z
- **Completed:** 2026-03-30T00:15:00Z
- **Tasks:** 2
- **Files modified:** 4

## Accomplishments

- `lib/vault.ts` : `initVault` crée maintenant des fichiers `gami-{id}.md` individuels pour chaque profil ; `addChild` crée `gami-{id}.md` pour le nouveau profil enfant au lieu d'appendre à `gamification.md`
- `hooks/useFarm.ts` et `hooks/useGamification.ts` : suppression de `GAMI_FILE = 'gamification.md'`, remplacement par un helper local `gamiFile(profileId)` — chaque lecture/écriture est scopée au fichier du profil actif
- `components/settings/SettingsGamification.tsx` : reset gamification boucle sur tous les profils et écrit un fichier `gami-{id}.md` par profil

## Task Commits

2. **Task 2: Migrer useFarm.ts, useGamification.ts, SettingsGamification.tsx** - `11c4de1` (feat)

## Files Created/Modified

- `lib/vault.ts` — helper `gamiFile()`, `initVault()` écrit `gami-{id}.md` par profil, `addChild()` crée `gami-{id}.md`
- `hooks/useFarm.ts` — `GAMI_FILE` → `gamiFile()`, `deductCoins`/`addCoins` per-profil avec filtrage
- `hooks/useGamification.ts` — `GAMI_FILE` → `gamiFile()`, `completeTask`/`openLootBox` per-profil avec filtrage
- `components/settings/SettingsGamification.tsx` — reset écrit `gami-{id}.md` par profil

## Decisions Made

- `gamiFile()` est défini localement dans chaque fichier plutôt qu'exporté depuis un module partagé — évite la dépendance circulaire `lib/vault.ts` ↔ `hooks/*.ts`
- `gamification.md` n'est pas supprimé dans cette phase — la migration backward-compatible (lecture de l'ancien fichier et split) est traitée dans Plan 02 (useVault.ts)
- `openLootBox` scrive uniquement le profil actif en `gami-{id}.md` ; le cas `family_bonus` qui doit modifier d'autres profils sera traité dans Plan 02 où tous les call sites de `useVault.ts` sont migrés
