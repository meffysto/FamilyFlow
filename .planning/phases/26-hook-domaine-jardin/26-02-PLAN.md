**Fonctions utilitaires (en dehors du hook) :**

1. `getMondayISO(date: Date): string` — retourne le lundi courant en format `'YYYY-MM-DD'` via `format(startOfWeek(date, { weekStartsOn: 1 }), 'yyyy-MM-dd')`.

2. `pickTemplate(weekIndex: number): ObjectiveTemplate` — retourne `OBJECTIVE_TEMPLATES[weekIndex % OBJECTIVE_TEMPLATES.length]`. Index rotatif deterministe (discretion Claude).

**Corps du hook `useGarden()` :**

1. **Consommation useVault** : `const { vault, gardenRaw, setGardenRaw, profiles } = useVault();`

2. **State local** : `const [isLoading, setIsLoading] = useState(false);`

3. **Derived state via useMemo** :
   - `gardenData = useMemo(() => parseGardenFile(gardenRaw), [gardenRaw])` — parse les donnees brutes
   - `activeProfileCount = useMemo(() => profiles.filter(p => p.role !== 'grossesse').length, [profiles])` — nombre profils actifs (Pitfall 6)
   - `currentTarget = useMemo(() => gardenData.currentTarget || computeWeekTarget(activeProfileCount), [gardenData, activeProfileCount])` — cible semaine (utilise la valeur persistee si disponible, sinon calcule)
   - `progress = useMemo(() => gardenData.contributions?.length ?? 0, [gardenData])` — nombre de contributions
   - `isGoalReached = useMemo(() => progress >= currentTarget, [progress, currentTarget])`
   - `currentTemplate = useMemo(() => { const id = gardenData.currentTemplateId; const found = OBJECTIVE_TEMPLATES.find(t => t.id === id); return found ?? pickTemplate(gardenData.pastWeeks?.length ?? 0); }, [gardenData])` — template actif
   - `weekHistory = useMemo(() => gardenData.pastWeeks ?? [], [gardenData])` — historique semaines

4. **useEffect generation objectif hebdomadaire** (per D-04, D-05, D-06, D-07) :
   ```typescript
   useEffect(() => {
     if (!vault) return;
     const currentMonday = getMondayISO(new Date());
     // Guard anti-boucle infinie + anti-double-generation (D-07, Pitfall 2)
     if (gardenData.currentWeekStart === currentMonday) return;

     (async () => {
       setIsLoading(true);
       try {
         const newTarget = computeWeekTarget(activeProfileCount);
         const template = pickTemplate((gardenData.pastWeeks?.length ?? 0) + (gardenData.currentWeekStart ? 1 : 0));

         // Archiver la semaine passee si elle avait un objectif (D-05)
         const updatedPastWeeks = [...(gardenData.pastWeeks ?? [])];
         if (gardenData.currentWeekStart) {
           const weekRecord: VillageWeekRecord = {
             weekStart: gardenData.currentWeekStart,
             target: gardenData.currentTarget ?? 0,
             total: gardenData.contributions?.length ?? 0,
           };
           updatedPastWeeks.push(weekRecord);
         }

         // Construire nouveau VillageData (contributions videes pour la nouvelle semaine)
         const newData: VillageData = {
           ...gardenData,
           currentWeekStart: currentMonday,
           currentTarget: newTarget,
           currentTemplateId: template.id,
           contributions: [],
           pastWeeks: updatedPastWeeks,
         };

         const newContent = serializeGardenFile(newData);
         await vault.writeFile(VILLAGE_FILE, newContent);
         setGardenRaw(newContent);
       } catch (e) {
         if (__DEV__) console.warn('useGarden — generation objectif:', e);
       } finally {
         setIsLoading(false);
       }
     })();
   }, [vault, gardenRaw, activeProfileCount]);
   ```
   **IMPORTANT** : Le commentaire dans le code doit expliquer l'idempotence iCloud (D-06) : "Formule deterministe — si deux profils triggerent simultanement, les deux ecritures produisent le meme resultat. Pas de corruption."

5. **addContribution** via useCallback (per D-09) :
   ```typescript
   const addContribution = useCallback(async (type: ContributionType, profileId: string): Promise<void> => {
     if (!vault) return;
     const contribution = {
       timestamp: new Date().toISOString(),
       profileId,
       type,
       amount: 1,
     };
     const newContent = await appendContributionToVault(vault, contribution);
     setGardenRaw(newContent);
   }, [vault, setGardenRaw]);
   ```
   Note : si `appendContributionToVault` retourne void au lieu de string, adapter en relisant le fichier apres ecriture : `await appendContributionToVault(vault, contribution); const updated = await vault.readFile(VILLAGE_FILE).catch(() => ''); setGardenRaw(updated);`

6. **claimReward** via useCallback (per D-08, D-09) :
   ```typescript
   const claimReward = useCallback(async (profileId: string): Promise<boolean> => {
     if (!vault || !isGoalReached) return false;

     // Lire le fichier gami du profil
     const gamiPath = `04 - Gamification/gami-${profileId}.md`;
     const content = await vault.readFile(gamiPath).catch(() => '');
     const farmData = parseFarmProfile(content);

     // Guard anti-double-claim (D-08)
     if (farmData.village_claimed_week === gardenData.currentWeekStart) return false;

     // Ecrire le flag anti-double-claim
     const profileName = profiles.find(p => p.id === profileId)?.name ?? profileId;
     const updated = { ...farmData, village_claimed_week: gardenData.currentWeekStart };
     await vault.writeFile(gamiPath, serializeFarmProfile(profileName, updated));

     return true;
   }, [vault, isGoalReached, gardenData, profiles]);
   ```
