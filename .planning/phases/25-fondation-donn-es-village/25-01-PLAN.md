**parseGardenFile(content: string): VillageData** — Parse le fichier jardin-familial.md :
- Utiliser `gray-matter` pour le frontmatter YAML (import matter from 'gray-matter')
- Champs frontmatter : `version` (number, defaut 1), `created` (string), `current_week_start` (string YYYY-MM-DD), `current_theme_index` (number, defaut 0), `reward_claimed` (boolean, defaut false)
- Parser la section `## Contributions` : chaque ligne `- YYYY-MM-DDTHH:mm:ss | profileId | type | amount` devient un VillageContribution
- Parser la section `## Historique` : chaque ligne `- YYYY-MM-DD | cible:N | total:N | claimed:bool` devient un VillageWeekRecord
- Si content est vide (''), retourner un VillageData par defaut : `{ version: 1, createdAt: '', currentWeekStart: '', currentThemeIndex: 0, rewardClaimed: false, contributions: [], pastWeeks: [] }`
- ATTENTION : `type` doit etre cast en `'harvest' | 'task'` — ignorer les lignes avec un type invalide
- ATTENTION : `amount` doit etre parseInt — ignorer les lignes avec un amount NaN

**serializeGardenFile(data: VillageData): string** — Serialise en markdown compatible Obsidian :
- NE PAS utiliser `matter.stringify()` (Pitfall 5 — lossy). Construire la string manuellement.
- Le format de sortie est un frontmatter YAML (delimiteurs triple-tiret) suivi de deux sections Markdown :
  - Frontmatter fields: version, created (= createdAt), current_week_start (= currentWeekStart), current_theme_index (= currentThemeIndex), reward_claimed (= rewardClaimed)
  - Section `## Contributions` avec chaque contribution formatee `- timestamp | profileId | type | amount`
  - Section `## Historique` avec chaque pastWeek formatee `- weekStart | cible:target | total:total | claimed:claimed`
- Sections toujours presentes meme si vides (facilite l'append)

**appendContribution(content: string, entry: VillageContribution): string** — Insere une ligne dans la section Contributions :
- Pattern IDENTIQUE a appendMuseumEntry dans lib/museum/engine.ts
- Format ligne : `- {timestamp} | {profileId} | {type} | {amount}`
- Trouver la section `## Contributions`, inserer AVANT la prochaine section `## ` (Pitfall 4 — ne jamais append en fin de fichier)
- Si section absente, la creer avant `## Historique` ou en fin de fichier
- Le timestamp dans l'entry est deja formate (l'appelant fournit le timestamp ISO sans Z)
