```typescript
export function useExpeditions() {
  // Retourne :
  return {
    dailyPool,           // ExpeditionMission[] — pool du jour (getDailyExpeditionPool)
    activeExpeditions,    // ActiveExpedition[] — expeditions en cours (depuis farmData)
    completedExpeditions, // ActiveExpedition[] — expeditions avec result non-undefined
    pendingResults,       // ActiveExpedition[] — complete mais result === undefined (a collecter)
    activeCount,          // number — nombre d'expeditions sans result
    canLaunch,            // boolean — activeCount < MAX_ACTIVE_EXPEDITIONS
    pityCount,            // number — expeditionPity depuis farmData
    launchExpedition,     // (mission: ExpeditionMission) => Promise<boolean>
    collectExpedition,    // (missionId: string) => Promise<{outcome, loot?}>
    dismissExpedition,    // (missionId: string) => Promise<void> — retire de la liste active
  };
}
```

**launchExpedition** :
1. Verifier canAffordExpedition (coins depuis profile.coins, harvestInventory depuis farmData)
2. Si non-affordable → Alert.alert('Ressources insuffisantes', 'Tu n\'as pas assez de feuilles ou de recoltes pour cette expedition.') → return false
3. Si activeCount >= MAX_ACTIVE_EXPEDITIONS → Alert.alert('Expeditions completes', 'Tu as deja 2 expeditions en cours. Attends le retour d\'une mission.') → return false
4. Alert.alert de confirmation (per D-07 / UI-SPEC copie exacte) :
   - Title: 'Confirmer l\'expedition ?'
   - Body: `Tu vas miser ${mission.costCoins} 🍃 + ${getExpeditionCostDescription(mission)} pour une expedition ${mission.durationHours}h.\n\nEn cas d'echec, toute la mise est perdue.`
   - Boutons: [{text:'Annuler', style:'cancel'}, {text:'Lancer', style:'destructive', onPress: async () => {...}}]
5. Dans onPress : Haptics.impactAsync(ImpactFeedbackStyle.Medium)
6. Deduire coins via addCoins(-mission.costCoins) depuis useVault
7. Lire farm file, muter harvestInventory (deduire cropIds × quantities), ajouter ActiveExpedition avec startedAt = new Date().toISOString()
8. Ecrire farm file via writeFile(farmPath, serializeFarmProfile(...))
9. Retourner true

**collectExpedition** :
1. Trouver l'expedition active par missionId qui est isExpeditionComplete && !result
2. Appeler rollExpeditionResult(exp.difficulty, pityCount)
3. Si outcome === 'failure' : incrementer expeditionPity += 1
4. Si outcome === 'success' || outcome === 'rare_discovery' : reset expeditionPity = 0
5. Si outcome === 'partial' : expeditionPity += 1 (partial compte comme echec partiel)
6. Appeler rollExpeditionLoot si outcome !== 'failure'
7. Si loot type === 'inhabitant' : ajouter dans mascotInhabitants
8. Si loot type === 'seed' : ajouter dans farmRareSeeds
9. Si loot type === 'booster' : activer le boost temporaire correspondant (buildingTurboUntil/growthSprintUntil/capacityBoostUntil avec +24h/+12h/+48h ISO)
10. Mettre a jour exp.result et exp.lootItemId/lootType dans activeExpeditions
11. Ecrire farm file mis a jour
12. Retourner { outcome, loot }

**dismissExpedition** :
1. Filtrer l'expedition hors de activeExpeditions (garde les autres)
2. Ecrire farm file mis a jour

Pattern suivi : meme readFile → modify → writeFile que useFarm.ts. Le farmPath est `04 - Gamification/farm-${profile.id}.md`. Utiliser useCallback sur tous les handlers.
