| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| `Spacing.xs` | 4px | Icon-text gap in tooltip, badge inner padding |
| `Spacing.md` | 8px | Tooltip vertical padding, catalog cell gap |
| `Spacing.xl` | 12px | Catalog cell internal padding, label margin |
| `Spacing['2xl']` | 16px | Tooltip horizontal padding, catalog grid padding, modal horizontal padding |
| `Spacing['3xl']` | 20px | Catalog section header margin |
| `Spacing['4xl']` | 24px | Modal content top padding, catalog row gap |

Exceptions:
- Building sprite render size: **72×72** (D-12) — not a spacing token, fixed pixel dimension for TileMapRenderer

- Touch targets on catalog tiles: minimum **44px tall** (iOS HIG) — achieved by catalog tile height via `minHeight: 44`
- Tooltip `maxWidth: 200` (pattern from AvatarTooltip Phase 29, inherited directly)

## Interaction Contracts

### Building sprite on village map (VILL-04)

- Visible only when the building is unlocked (`unlockedBuildings` contains its `buildingId`)
- Tap triggers `BuildingTooltip` with auto-dismiss 2500ms (same as AvatarTooltip)
- On tap: `Haptics.selectionAsync()` (per CLAUDE.md convention)
- Animation: fade-in opacity 0→1 over 180ms + translateY -4→0 (identical to AvatarTooltip enter)
- `pointerEvents="none"` on tooltip itself (event passthrough to map below)
- No tap-to-navigate: tooltip is terminal interaction (D-13 — deferred in `<deferred>`)

### Building tooltip content

- Template: `"[Label FR] — Débloqué à {palier} feuilles familiales"`
- Example: `"Café — Débloqué à 1 500 feuilles familiales"`
- `numberOfLines={1}` with maxWidth 200 (same constraint as AvatarTooltip, D-14)
- Font: `FontSize.sm` (14px), `FontWeight.normal` (400), `colors.text`

### Catalog modal (VILL-06)

- Opens via header button: `MaterialCommunityIcons` name `"castle"` or `"home-city"` (Claude's discretion, pick whichever renders cleanest at 22px). Placed on right side of village.tsx header.
- Modal: `presentation: 'pageSheet'` + `enableDismissOnClose` (drag-to-dismiss, CLAUDE.md standard)
- 2-column grid, 8 tiles (4 rows)
- Each tile:
  - **Unlocked**: sprite 96×96 full color, label FR (`FontSize.label` 13px semibold), "Débloqué" text in `colors.success`
  - **Locked**: sprite 96×96 with `tintColor: colors.textMuted` + `opacity: 0.4` (silhouette), label FR visible, `"À {palier} feuilles"` in `colors.textMuted` (`FontSize.caption` 12px), progression `"{current}/{target}"` in `colors.textFaint` (`FontSize.caption`)
  - **"Nouveau" badge**: absolute-positioned top-right of tile, shown when building is newly unlocked and not yet seen. Badge: `colors.successBg` background, gold star `#FFD700`, text `"Nouveau"` at `FontSize.micro` (10px) `FontWeight.semibold`.
  
- Tile tap behavior:
  - **Unlocked tile**: subtle spring scale pulse (`withSpring` damping 12 stiffness 180, scale 1→1.05→1), no navigation (D-20)
  - **Locked tile**: toast message `"Encore {remaining} feuilles"` (use existing toast from `ToastProvider`) + `Haptics.impactAsync(ImpactFeedbackStyle.Light)` (D-20)

### "Nouveau" badge lifecycle

- On catalog mount: compare `gardenData.unlockedBuildings` list against `lastSeenBuildingsUnlock` timestamp from `expo-secure-store` key `"village_buildings_seen_at"`
- Buildings whose unlock timestamp is after `lastSeenBuildingsUnlock` are flagged as new
- Badge entrance: `withSpring` scale 0→1, damping 12 stiffness 180 (D-19)
- On catalog close: write `new Date().toISOString()` to `"village_buildings_seen_at"` via `SecureStore.setItemAsync` (D-19)

### Unlock detection (background)

- In `useGarden.ts`: compute `familyLifetimeLeaves = profiles.reduce((sum, p) => sum + (p.points ?? 0), 0)` (D-05)
- On each change to `familyLifetimeLeaves` OR on mount: check each `BUILDINGS_CATALOG` entry whose palier ≤ `familyLifetimeLeaves` AND not already in `unlockedBuildings`
- If new threshold crossed: call `appendBuilding(buildingId, palier, timestamp)` — append-only to `jardin-familial.md` section `## Constructions` (D-21, D-23)
- Pattern: identical to `appendContribution` in `lib/village/parser.ts`
- Idempotent: check `unlockedBuildings.some(b => b.buildingId === id)` before writing (D-23)

## Animation Specifications

All animations via `react-native-reanimated` exclusively (CLAUDE.md constraint).

| Interaction | Animation | Config |
|-------------|-----------|--------|
| Tooltip enter | `opacity` 0→1 + `translateY` -4→0 | `withTiming` 180ms |
| Tooltip exit (auto) | `opacity` 1→0 + `translateY` 0→-4 | `withTiming` 150ms |
| "Nouveau" badge enter | `scale` 0→1 | `withSpring` damping 12 stiffness 180 |
| Unlocked tile tap | `scale` 1→1.05→1 | `withSpring` damping 12 stiffness 180 |
| Building appear on map | `opacity` 0→1 on first render | `withTiming` 300ms |

Spring constant: `const SPRING_CATALOG = { damping: 12, stiffness: 180 }` — defined as module constant per CLAUDE.md convention.
