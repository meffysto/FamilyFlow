**Implémentation fallback** : iOS peut bloquer l'autoplay programmatique sans interaction utilisateur (RESEARCH.md §272-275, A1 et Pitfall 3). Si Task 1 a montré que l'audio ne démarre pas, ajouter un état détectant l'échec et présentant un gros bouton play centré.

**Stratégie** : timer de 2 secondes après tentative autoplay. Si `isPlaying` reste `false`, afficher un overlay avec bouton play centré.

**Modification `app/story/[id].tsx`** : ajouter un état `audioStarted` et un ref pour mesurer si l'autoplay a effectivement démarré.

**OU** plus simple — modifier `StoryPlayer.tsx` pour exposer un callback `onAutoplayFailed?: () => void` qui se déclenche si après 2s `isPlaying === false` malgré `autoplay === true`. La route consomme ce callback pour afficher un fallback.

**Implémentation minimale** dans `StoryPlayer.tsx` :

```typescript
// Dans Props :
/** Callback déclenché si autoplay demandé mais audio pas démarré après 2s (iOS audio policy). */
onAutoplayFailed?: () => void;

// Dans la signature destructurée :
onAutoplayFailed,

// Ajouter useEffect après le useEffect autoplay (Plan 50-02 Task 2) :
useEffect(() => {
  if (!autoplay) return;
  if (!autoplayTriggeredRef.current) return; // pas encore tenté
  const timeout = setTimeout(() => {
    if (!isPlaying && onAutoplayFailed) {
      if (__DEV__) console.warn('[StoryPlayer] autoplay iOS bloqué — fallback bouton');
      onAutoplayFailed();
    }
  }, 2000);
  return () => clearTimeout(timeout);
}, [autoplay, isPlaying, onAutoplayFailed]);
```

**Implémentation route `app/story/[id].tsx`** : ajouter un overlay fallback affiché si `onAutoplayFailed` se déclenche.

```typescript
const [autoplayBlocked, setAutoplayBlocked] = useState(false);

// Dans le JSX :
<FullscreenStoryReader
  histoire={histoire}
  voiceConfig={storyVoice.voiceConfig}
  elevenLabsKey={storyVoice.elevenLabsKey}
  fishAudioKey={storyVoice.fishAudioKey}
  autoplay
  onAutoplayFailed={() => setAutoplayBlocked(true)}
  onClose={() => router.replace('/(tabs)/journal')}
/>
{autoplayBlocked && (
  <View style={[styles.fallbackOverlay, { backgroundColor: 'rgba(0,0,0,0.55)' }]} pointerEvents="none">
    <Text style={[styles.fallbackText, { color: colors.background }]}>
      Touche le bouton lecture pour démarrer l'audio
    </Text>
  </View>
)}
```

**Note** : `pointerEvents="none"` pour ne pas bloquer le tap sur le bouton play du `StoryPlayer` lui-même. C'est juste un hint visuel.
