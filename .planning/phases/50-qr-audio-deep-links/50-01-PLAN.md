Tant qu'aucun fichier `apple-app-site-association` n'est servi sur ce domaine, iOS ignorera silencieusement l'entrée — pas d'erreur runtime, juste pas de matching HTTPS Universal Link (RESEARCH.md §54-72).

```json

  "supportsTablet": true,
  "bundleIdentifier": "com.familyvault.dev",

}
```

**Justification du nom** : `placeholder.familyflow.app` rend l'intention explicite (placeholder, non actif). Lors de la migration future Universal Links, ce sera remplacé par un vrai domaine type `familyflow.app` ou `links.familyflow.app` après acquisition du domaine + hébergement AASA.
