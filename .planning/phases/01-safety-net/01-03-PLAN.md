    Dans app/_layout.tsx:
    - Ajouter en HAUT du fichier (avant tout autre import si possible, sinon juste apres les imports React):
      ```typescript
      import * as Sentry from '@sentry/react-native';

      Sentry.init({
        dsn: process.env.EXPO_PUBLIC_SENTRY_DSN,
        tracesSampleRate: 0.2,
        enabled: !__DEV__,
        environment: __DEV__ ? 'development' : 'production',
      });
      ```
    - L'init DOIT etre au niveau module (pas dans un useEffect ou une fonction composant)
    - Trouver le composant par defaut exporte (probablement `export default function RootLayout` ou equivalent)
    - Wrapper l'export: `export default Sentry.wrap(RootLayout);`
    - Si l'export est deja une function expression anonyme, extraire dans une variable nommee d'abord

    NE PAS creer de fichier metro.config.js — le plugin `@sentry/react-native/expo` (ajoute dans app.json plugins) gere automatiquement la configuration metro pour les source maps via le systeme de config plugins d'Expo. Ce comportement est documente dans la section "Expo" de https://docs.sentry.io/platforms/react-native/manual-setup/expo/ : le plugin modifie le metro config en interne sans necessiter de fichier metro.config.js explicite. Cela differe du setup manuel (non-plugin) ou metro.config.js est requis.

    NE PAS hardcoder le DSN. Utiliser `process.env.EXPO_PUBLIC_SENTRY_DSN`.
