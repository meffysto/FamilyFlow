Auto-placement — chaîne de décision

Principe : toute tâche du jour est slottée. Pas de purgatoire "À placer". Si l'app fait un choix, il est marqué visible auto — tap pour changer, long-press pour renvoyer la tâche au backlog (mode Liste).

  1. Décision utilisateur explicite — timeSlot dans le markdown (emoji ☀️🍽️☕🌙) → respecté absolument.
  2. Heure existante — reminderTime ou dueDate.time → mappé via tableau ci-dessous.
  3. Historique de complétion (≥2 occurrences passées) — l'app observe quand tu coches habituellement cette tâche, place dans le slot dominant. Tag auto affiché.
  4. Source file structurée — fichier nommé "Routine matin.md" → slot matin. Tag auto affiché.
  5. Next-fit ≤75% — la tâche entre dans le premier slot du jour dont la charge reste ≤ 75% (3h sur 4 pour aprem). Tag auto affiché. Aucun slot ne dépasse 75% par défaut → buffer anti-Parkinson garanti.

Vault Obsidian : l'app n'écrit l'emoji slot dans le markdown que si l'utilisateur a validé ou modifié le placement (verrou). Sinon, l'auto-placement vit en mémoire/cache uniquement. Le vault reste lisible humain, le markdown n'enregistre que les décisions humaines.

Gamification : zéro impact. Le champ timeSlot est purement organisationnel — aucun calcul XP, streak, ou quête coop ne le lit. Seul CACHE_VERSION bumpé pour le nouveau champ optionnel dans la cache.
