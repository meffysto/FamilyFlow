## Global Settings

| Parameter | Value | Rationale |
|-----------|-------|-----------|
| `--ar` | `3:4` | Portrait mobile-friendly, good for vertical tree growth |
| `--s` | `750` | High stylization for painterly quality without losing structure |
| `--v` | `6.1` | Best photorealistic-to-artistic bridge in current MJ |
| `--q` | `1` | Standard quality (2 if budget allows for legendary stage) |
| `--chaos` | `0-10` | Low chaos to maintain consistency across batch |
| `--style` | `raw` | Optional: use if results are too "MJ-styled", raw gives more prompt fidelity |

For DALL-E 3: request "watercolor illustration" style, "natural" mode (not "vivid"), square or portrait format.

## Batch Generation Guide

### Recommended Workflow

1. **Generate the Style Anchor first** as a standalone image. Use it as a `--sref` (style reference) in Midjourney v6.1 for all subsequent generations.

2. **Generate in species order**, completing all 6 stages of one species before moving to the next. This maintains internal species consistency.

3. **For each species**, generate Stage 1 first, then use `--sref` from that result plus the global style reference to anchor subsequent stages.

4. **Legendary stages (Stage 6)** use `--q 2` for extra detail on the magical elements. These are the hero images and benefit from the higher compute.

### Midjourney Batch Syntax

To generate all 6 stages of a species efficiently, you can use Midjourney's repeat feature:

```
/imagine [STYLE ANCHOR] [Stage-specific prompt] --ar 3:4 --s 750 --v 6.1 --repeat 4
```

Generate 4 variations per prompt, then select the best from each for consistency.

### Consistency Tips

- Use `--sref [URL]` with the first approved seed/sprout image on every subsequent prompt
- Keep `--s 750` constant (except legendary at 800)
- Keep `--chaos` at 0-10 for predictable results
- If a result drifts from the style, add `--sw 100` (style weight) to strengthen the style reference
- Generate at `--ar 3:4` for mobile display, crop later if needed
- Consider using `--cref` (character reference) with Stage 1 of each species to maintain species identity across stages

### Post-Processing Notes

- All images will need background removal for in-app use (transparent PNG)
- Keep the soil/ground base when removing background — it anchors the tree
- Export at 2x resolution for retina mobile displays (600px effective width)
- Apply consistent color correction across each species set
- Legendary stage glow effects may need subtle enhancement in post
